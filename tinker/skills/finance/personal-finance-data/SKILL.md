## Architecture

See `references/accounts.md` for the full account registry with row counts, date ranges, and balances.

- **SQLite** at `finance.db` — golden source, committed to git
- **Bank accounts** (barclays, hsbc, amex_gold, amex_ba, hsbc_credit, lloyds) — per-transaction with running balance column, imported via `import.py` from CSV/OFX in `inbox/`
- **Vanguard** (ISA + Pension) — parsed from XLSX statements via `vanguard_import.py`, stored in `vanguard_isa_cash`, `vanguard_isa_investment`, `vanguard_pension_cash`, `vanguard_pension_investment`
- **Mortgage** (NatWest 2019-2024, Barclays 2024-present) — monthly amortization in `mortgage` table, generated by `mortgage.py` from `mortgage_config.py`
- **Balance sheet** — `balance_sheet(date, account, amount, kind)` table, monthly snapshots built by `build_balance_sheet.py`
- **Investment holdings** — `investment_holdings` table, current units × price computed by `compute_holdings.py`
- **Views**: `v_combined` (all cash-flow accounts), `v_balance_sheet` (latest snapshot only)

## Adding a new account type

1. **Write parser** — parse the source format (CSV, XLSX, OFX) into a list of dicts
2. **Create DB tables** in `init_schema.py` with dedup indexes
3. **Write import script** (or extend import.py) that reads raw file → inserts to table
4. **Add to balance_sheet** in `build_balance_sheet.py` — decide: asset or liability, cash balance or computed value
5. **Re-run pipeline**: import → compute_holdings → build_balance_sheet

## Investment value computation

**The simple rule (user prefers this): cumulative units × latest trade price.**

ISA: `SUM(quantity) FROM vanguard_isa_investment WHERE date <= month × last price <= month`
Pension: same from `vanguard_pension_investment`

No forward-fill, no interpolation, no market data — just what's in the Vanguard statement.

## Balance sheet building

`build_balance_sheet.py` walks every month from earliest data to current month and records each account's value. Monthly granularity (matches mortgage + statements). Accounts only appear in months where they have data (no pre-fill with zeros).

**Current accounts tracked:**
- Barclays Current, HSBC Current (from `balance` column)
- HSBC Credit Card (from `balance` column, stored as liability)
- Vanguard ISA Cash/Investment, Vanguard Pension Cash/Investment
- NatWest Mortgage (2019-11 to 2024-10), Barclays Mortgage (2024-11 to now)

**NOT tracked (no balance data):** AMEX Gold, AMEX BA, Lloyds (closed)

## Common pitfalls

- **Investment value is NOT cash balance** — pension shows £0.01 cash but £6,456 in investments. Always use investment_holdings or balance_sheet, not vanguard_pension_cash.balance.
- **AMEX has no balance column** — can't use for balance sheet. Only transaction amounts.
- **Mortgage table has future projections** — always cap at `date('now')` or use the balance_sheet table which already does this.
- **~/.hermes/profiles/tinker/home overrides HOME** — use absolute paths, not `~`, when running python scripts from terminal tool.
- **User prefers simple over complex** — if you find yourself explaining forward-fill, interpolation, or market data feeds, you're over-engineering. Just multiply units × price.

## Quick verification

```bash
# Latest balance sheet snapshot
sqlite3 -column -header finance.db "SELECT * FROM balance_sheet WHERE date=(SELECT MAX(date) FROM balance_sheet) ORDER BY kind, account;"

# Net worth trend (annual)
sqlite3 -column -header finance.db "
SELECT date, ROUND(SUM(CASE WHEN kind='asset' THEN amount ELSE amount END)) AS net_worth
FROM balance_sheet WHERE date LIKE '%-01-01' OR date LIKE '%-06-01'
GROUP BY date ORDER BY date;"
```
