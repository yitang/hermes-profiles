# The Knowledge Archive vs Practice Gap

## Pattern

User builds a knowledge repository (git repo, obsidian vault, org-mode collection) by dumping notes from sessions, tutorials, and research. Goal: "relies less on Google, more on memorizing syntax and tool capabilities." Problem: never studies the notes. Repository becomes a dump where relevant information gets buried.

## Diagnosis Questions

1. **"What's actually blocking you?"** — two main causes:
   - **Structural**: Notes are so unstructured it takes more effort to find stuff than Googling (the "find by nature" problem)
   - **Ritual**: No regular time carved out; notes get pushed by work tasks

2. **"Do you use this for day-to-day work?"** — honest answer reveals whether the repo is still infrastructure or has become a graveyard.

3. **"What's your mental model of these tools?"** — determines which restructuring strategy (category-first, hierarchy tree, index-driven, mixed) will feel natural.

## Intervention

Restructuring alone won't fix an engagement problem. Best approach:
1. **Preserve everything** — keep old/outdated notes in place. User preference: signal extraction from noise, not curation-by-deletion. Use index cards and recency filters to surface what's relevant without destroying history.
2. **Structure for browsing** — add auto-generated index cards that cluster by topic keyword (filename + tags) and order by recency. Org-roam backlinks enable graph-based navigation as a secondary discovery path.
3. **Pair structure with study plan** — a "study the restructured PKB" session should be scheduled immediately after restructuring, while engagement is high.
